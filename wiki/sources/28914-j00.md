## Summary

3GPP TR 28.914 is the Rel-19 study report that investigated additional enhancements for intent-driven management beyond the Rel-18 study (TR 28.912). It studied 20 use cases covering enhanced radio service/network expectations, implicit intent report subscription, graded service guarantee, intent lifecycle documentation, natural language intent translation, intent fulfilment enablers, intent degradation with preference weighting, utility function support, feasibility check with intentMgmtPurpose, intent exploration, negotiation on fulfilment outcomes, guidelines for vendor extensions, network maintenance intent, and communication service intent.

## Key Claims

- Enhanced RadioServiceExpectation with PLMNInfoContext, maxNumberofUEsTarget, and SchedulingTimeContext for scheduled radio service delivery
- RadioNetworkExpectation enhancements: ulFrequencyContext, rATContext, ActiveUEsTarget, PRBsTarget, InterRATHandoverTargets, SchedulingTimeContext, cellContext, timeBasedAreaContext for RAN energy saving, scheduled events, UAV support, MOCN
- IntentReportControl dataType proposed for implicit intent report subscription with customized requirements (reportRecipientAddress, expectedReportTypes, reportingConditions, reportingTargets, observationPeriod)
- Intent handling state management: state diagram with ACKNOWLEDGED, COMPLIANT, FULFILLED, DEGRADED, SUSPENDED, FULFILLMENT_FAILED, TERMINATED and transition table
- PreferenceWeight attribute proposed for IntentExpectation and ExpectationTarget to support intent degradation based on expectation preference
- Intent Utility Function: mathematical expression with variables, weights, function type, and result; satisfaction index as integer [0,100] to quantify consumer's relative preference
- IntentMgmtPurpose attribute (FEASIBILITYCHECK, EXPLORE, FULFILMENT) proposed to differentiate intent management operations
- IntentExploreResult dataType proposed for intent exploration — MnS producer provides recommended best values for targets/contexts via simulation/evaluation
- Intent negotiation during fulfilment phase: fulfillable outcomes reporting, alternatives with relative impacts, consumer preference via utility function or satisfaction index, recommended changes for unfulfillable intents
- Guidelines for using generic IntentExpectation dataType for vendor-defined scenarios not standardized in TS 28.312
- IntentExpectation for network maintenance: ExpectationVerb, templateSupportRequirementContext, maintenance-related contexts and targets
- Natural language intent translation studied but not recommended for R19 normative work
- Communication service intent expectation proposed with NetworkSlice as objectType
