## Summary

3GPP TS 28.622 defines the Generic Network Resource Model (NRM) Integration Reference Point (IRP) Information Service — the foundational information model for all 3GPP management specifications. It defines the core IOCs (Top, TopX, SubNetwork, ManagedElement, ManagedFunction, ManagementNode, MeContext, MnsAgent, etc.) and common data types (GeoArea, TimeWindow, SchedulingTime, PlmnId, IpAddr, etc.) that are inherited and reused by domain-specific NRM specifications (TS 28.541 for 5G NRM, TS 28.556 for Policy, TS 28.312 for Intent, etc.).

## Key Claims

- **Top IOC** is the root of the IOC inheritance hierarchy for all TS conformant to TS 32.102; it inherits from TopX (objectClass, objectInstance) and Top_ (from TS 28.620 UIM)
- **TopX IOC** provides the mandatory attributes objectClass and objectInstance — the minimal identification for any managed object
- **SubNetwork IOC** represents a set of managed entities; the root SubNetwork instance contains all others; inherits from Domain_ and TopX
- **ManagedElement IOC** represents telecom equipment or TMN entities; may contain 1..N ManagedFunction instances; inherits from ManagedElement_ and TopX
- **ManagedFunction IOC** is abstract (provided for sub-classing); represents functional aspects contained in ManagedElement; inherits from Function_ and TopX
- **MnsAgent IOC** represents MnS producers within the Service Based Management Architecture (SBMA); inherits from Top_ and TopX
- **VsDataContainer IOC** allows vendor-specific data to be attached to any IOC (contained by Top)
- **NtfSubscriptionControl IOC** manages notification subscriptions with scope, filter, and recipient address; referenced by IntentReport subscription mechanism
- **ThresholdMonitor IOC** monitors performance metric thresholds with hysteresis and directional triggering
- **HeartbeatControl IOC** controls heartbeat notification emission for monitoring communication channels
- **TraceJob IOC** represents trace/MDT control and configuration parameters
- **PerfMetricJob IOC** represents performance metric collection jobs
- **MnsRegistry IOC + MnsInfo IOC** represent MnS service registry for capability discovery
- Common data types: GeoArea, TimeWindow, GeoCoordinate, SchedulingTime, TimeInterval, PlmnId, IpAddr, Host, PLMNInfo, AreaScope, Tai, NpnId, Scope, NodeFilter, VnfParameters, PeeParameters, TraceReference, ProcessMonitor
- Supports FNIM (Federated Network Information Model) concept from TS 32.107/28.620
- Release 20 version — latest, supersedes the Rel-14 IRP framework version for SBMA deployments

## Key Quotes

> "All information object classes defined in all TS that claim to be conformant to TS 32.102 [2] and support the Federated Network Information Model (FNIM) concept shall inherit from Top." — Section 4.3.29.1

> "All information object classes defined in all TS that claim to be conformant to TS 32.102 [2] shall inherit from TopX." — Section 4.3.8.1

> "The present document specifies the Generic network resource information that can be communicated between an MnS producer and MnS consumer for telecommunication network management purposes, including management of converged networks and networks that include virtualized network functions." — Section 1
